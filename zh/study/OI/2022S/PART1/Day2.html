    # Day2 数据结构2

    # ST表（稀疏表）

    ST 表 - OI Wikiopen in new window

    例题

    给定 nnn 个数，有 mmm 个询问，对于每个询问，你需要回答区间 [l,r][l,r][l,r] 中的最大值。

    1. n,m≤105n,m \le 10^5n,m≤105；
    2. n≤105,m≤107n \le 10^5,m\le10^7n≤105,m≤107。
    • 暴力：每次都对区间 [l,r][l,r][l,r] 扫描一遍，求出最大值，这显然会超时。
    • 分块：复杂度 O(mn)O(m\sqrt{n})O(mn​) ，也许能够通过1，但不可能通过2。
    • 单调栈+二分：复杂度 O(mlog⁡n+fsort(m))O(m \log n+f_{sort}(m))O(mlogn+fsort​(m)) ，就算使用最快的基数排序或挂链排序也不能保证通过2。

    ST表（稀疏表）是一种基于倍增思想，可以做到 O(nlog⁡n)O(n\log n)O(nlogn) 预处理， O(1)O(1)O(1) 回答每个区间询问的数据结构。但是不支持修改操作。

    基于倍增思想，我们考虑如何求出区间最大值。

    为了更快地查询区间最大值，我们需要提前预处理出一些区间的答案。 长度为 nnn 的序列上区间的个数为 n(n+1)/2n(n+1)/2n(n+1)/2 ，时间和空间都不允许把他们预处理出来。 于是退而求其次，预处理出所有长度为 2i2^i2i 的区间的最大值，其中区间 [x,x+2i)[x,x+2^i)[x,x+2i) 的最大值记作 st[x][i]。 显然地，st[x][0] = val[i]，st[x][i+1] = max(st[x][i], st[x+(1<<i)][i])。 预处理复杂度 O(nlog⁡n)O(n \log n)O(nlogn) 。

    下面考虑如何通过预处理出的这些区间快速的到任意区间的最大值。 对于任意区间 [l,r][l,r][l,r]，总能找到不超过两个长度为 （记其中指数部分为 ）的区间： 和 ，恰好完全覆盖 （允许重复）。 又因为区间最大值是一个可重复贡献问题（即：数 被统计的次数对答案无影响），所以可以将这两个区间的最大值简单合并，得到区间 的最大值。 单次查询复杂度 。
