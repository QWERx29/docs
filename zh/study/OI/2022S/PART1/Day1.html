    # 复杂度分析方法

    嵌套循环：相乘；顺序结构：相加（在大 OOO 意义下为取最大值）

    # 调和级数

    考虑如下代码的时间复杂度：

    for(int i=1;i<=n;i++)
    for(int j=i;j<=n;j+=i){
        // do something in O(1)
    }

    直接将两层循环的（最大）次数相乘，得到复杂度 O(n2)O(n^2)O(n2) 。这确实是一个上界，但不是一个严格渐进。

    对于确定的 iii ，内层循环的次数约为 ni\frac{n}{i}in​，以下的调和级数求和公式给出了一个更好的上界 O(nlog⁡n)O(n \log n)O(nlogn)（同时也是严格渐进）：

    # 均摊复杂度

    以 KMP 算法中计算 next 数组的代码为例：

    nex[0]=-1,nex[1]=0;
for(int i=2,j=0;i<=m;++i){
    while(j!=-1&&t[i]!=t[j+1])j=nex[j];
    nex[i]=++j;
}

    外层循环次数为 ，内层循环在最坏情形下次数也可能达到 mmm。

    但众所周知 KMP 算法的复杂度是 O(n+m)O(n+m)O(n+m) 的。

    这是因为以上代码中的变量 j 只会自增 mmm 次，而每次内层循环都会将 j 减小，因此内层循环的次数之和不会超过 mmm。

    于是以上代码的总复杂度为 O(m)O(m)O(m)，内层循环的均摊复杂度为 O(m)m\frac{O(m)}{m}mO(m)​。

    # * 势能分析

    势能分析是一种求均摊复杂度上界的方法。

    求均摊复杂度，关键是表达出先前操作对当前操作的影响。势能分析通过定义一个势函数来表达此种影响。

    在上面 KMP 的例子中，取 f(statei)=jif(\text{state}_i) = j_if(statei​)=ji​ 就是一个势能函数。

    设 tit_iti​ 为外层循环 的时间开销，则 ti≤1+(ji−1−ji+1)=ji−1−ji+2t_i \leq 1 + (j_{i-1} - j_i + 1) = j_{i-1} - j_i +2ti​≤1+(ji−1​−ji​+1)=ji−1​−ji​+2 。

    记 pi=ti+f(statei)−f(statei−1)=ti+ji−ji−1≤2p_i = t_i + f(\text{state}_i) - f(\text{state}_{i-1}) = t_i + j_i - j_{i-1} \leq 2pi​=ti​+f(statei​)−f(statei−1​)=ti​+ji​−ji−1​≤2 。

    于是 2m≥∑i=1mpi=∑i=1mti+f(statem)−f(state0)≥∑i=1mti2m \geq \sum_{i=1}^{m}{p_i} = \sum_{i=1}^{m}{t_i} + f(\text{state}_m) - f(\text{state}_0) \geq \sum_{i=1}^{m}{t_i}2m≥∑i=1m​pi​=∑i=1m​ti​+f(statem​)−f(state0​)≥∑i=1m​ti​

    于是总复杂度为 O(m)O(m)O(m) ，均摊复杂度为 O(pi)=O(1)O(p_i)=O(1)O(pi​)=O(1) 。

    # 常数问题

    由渐进复杂度得出的时间开销并不能充分地刻画程序的实际运行时间。一些渐进复杂度不优的程序反而在实际运行比渐进复杂度较优的程序快。在比较极端的情况下，甚至出现了：

    O(n2)O(n^2)O(n2) 过百万，暴力踩标算。

    相同渐进复杂度的程序运行的快慢可以用常数来定性地刻画。（注意常数在这里是一个定性的概念，而不是一个能够明确定义的数值）

    定性地说，渐进复杂度相同的程序：

    • 操作越多，耗时越长，常数越大
    • 对于单条操作，大致有：
      • 逻辑运算 / 位运算 < 加减法 < 乘法 << 除法 / 取余
      • int 类型运算 < long long 类型运算 < 浮点类型运算
      • 计算 / 赋值语句 < 控制语句
    • 与递推程序相比，递归程序的栈操作会带来额外的时间开销

    在 CSP / NOIP 阶段，一般无需考虑程序常数过大造成的运行超时问题；但如果常数优化较好，也有可能收获暴力程序通过额外的测试点的意外之喜~~（最好不要对此抱有什么期望）~~

    # 堆

    堆是一种树形数据结构，每个结点的键值都大于等于（或小于等于）其父节点的键值。

    按堆顶（即树根）键值是全堆的最大或最小值，堆可以分为大根堆与小根堆。

    所有堆支持的三种基本操作是插入元素、查询堆顶、删除堆顶。

    STL中的容器适配器 std::priority_queue 在默认情况下是使用 std::vector 存储的二叉大根堆。

    # 手写二叉堆

    二叉堆 - OI Wikiopen in new window

    最简单的堆是二叉堆，其形态是一棵完全二叉树。二叉堆可以直接使用数组进行存储，结点 x 的左右儿子为 2x = x<<1 与 2x+1 = x<<1|1。

    堆的应用范围较广，除了在 Dijkstra 算法中维护最近结点外，还广泛应用于在贪心题目中维护最优决策。

    #2523 Heap A

    贪心策略：若桌面空间不足，则桌面上下一次使用最晚的工具放回工具箱。

    感性理解正确性：让下一次使用较早的工具留在桌面上更有可能使之后几次不需要取新工具。
